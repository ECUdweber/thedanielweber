Overview of React

React was created at Facebook and is a UI Library that focuses on interactive, stateful and reusuable elements. It shines when building large applications with data that changes over time.

React makes it easy to display data and also automatically updates the UI when the underlying data changes.

What it offers

Reasons to use React

It's simple. All you need to do is express how your app should look at any given point in time, and React automatically manages all UI updates when your underlying data changes.

React forces you to think in terms of reusable components (these are all you can create with react). This allows you to add and remove components much easier than other "frameworks". React is much more than another library, it is a new way of thinking. See reacts github page for help thinking in terms of react.

Has Virtual DOM, which allows for extremely fast rendering. Only renders portions of the DOM that were affected to get to a new state.

Setting things up

When you set up your page, you need to include both react.js and JSXTransformer.js. You can then write your component in a script tag with type set to text/jsx: 


<!DOCTYPE html>
<html>
  <head>
    <script src="build/react.js"></script>
    <script src="build/JSXTransformer.js"></script>
  </head>
  <body>
    <div id="app-entry-point"></div>
    <script type="text/jsx">
      // React code will go here
    </script>
  </body>
</html>

Components connect to a dom element. In this instance, we can use the div with id of "app-entry-point" as it's entry point.

Basic Concepts

The most basic elements in React are called components. Components are simply state-machines. React thinks of UIs as simple state machines. By thinking of a UI as being in various states and rendering those states, it's easy to keep your UI consistent.

An example follows: 


<script type="text/jsx">
    /** @jsx React.DOM */
    React.render(
        <h1>Hello, world!</h1>,
        document.getElementById('app-entry-point')
    );
</script>

Components connect to a dom element. In this instance, we can use the div with id of "app-entry-point" as it's entry point.

Every component has a state object and a props object. State is set using the setState method and calling it triggers UI updates. 


<script type="text/jsx">
    /** @jsx React.DOM */
    var CounterComponent = React.createClass({
        getInitialState: function(){
            return {
                count: 5
            }
        },
        render: function(){
            return (
                <h1>{this.state.count}</h1>
            )
        }
    });
</script>

The scripting language shown is called JSX and is a Javascript XML syntax transform.

Events

React has an events system. The events are attached as properties of components and can trigger methods. To build off the example above, let's increment the count on click: 


<script type="text/jsx">
/** @jsx React.DOM */

var Counter = React.createClass({
  incrementCount: function(){
    this.setState({
      count: this.state.count + 1
    });
  },
  getInitialState: function(){
     return {
       count: 0
     }
  },
  render: function(){
    return (
      <div class="my-component">
        <h1>Count: {this.state.count}</h1>
        <button type="button" onClick={this.incrementCount}>Increment</button>
      </div>
    );
  }
});

React.render(<Counter/>, document.getElementById('entry-point'));

</script>

Using Components

When you use the render function, the first argument is the component we want to render, and the second is the DOM node it should be connected to. Use the createClass method to create custom component classes. That takes an object of specifications as an argument as demonstrated below: 


var TestComponent = React.createClass({
    render: function(){
        return (
            <h1>Hello, world!</h1>
        );
    }
});

// Now, we can render this component
React.render(
    <TestComponent/>,
    document.getElementById('someDiv')
);

When you create a component class with React.createClass(), provide a specification object that contains a render method. Additionally, you can contain other lifecycle methods described here. 


var TestComponent = React.createClass({
    render: function(){
        return (
            <h1>Hey {this.props.name}!</h1>
        );
    }
});

React.render(<TestComponent name="Genius" />, document.getElementById('someDiv'));

React.createClass() also contains other Lifecycle Methods:.

  • Mounting: componentWillMount - Invoked once, both on the client and server, immediately before the initial rendering occurs.
  • Mounting: componentDidMount - Invoked once, only on the client, immediately after the initial rendering occurs.
  • Updating: componentWillReceiveProps - Invoked when a component is receiving new props. This method is not called for the initial render.
  • Updating: shouldComponentUpdate - Invoked before rendering when new props or state are being received. This method is not called for the initial render or when forceUpdate is used.
  • Updating: componentWillUpdate - Invoked immediately before rendering when new props or state are being received. This method is not called for the initial render.
  • Updating: componentDidUpdate - Invoked immediately after the component's updates are flushed to the DOM. This method is not called for the initial render.
  • Unmounting: componentWillUnmount - Invoked immediately before a component is unmounted from the DOM.

It also contains several specs (other than render):

  • getInitialState - Invoked once before the component is mounted. The return value will be used as the initial value of this.state.
  • getDefaultProps - Invoked once and cached when the class is created. Values in the mapping will be set on this.props if that prop is not specified by the parent component (i.e. using an in check).
  • propTypes - The propTypes object allows you to validate props being passed to your components.
  • mixins - The mixins array allows you to use mixins to share behavior among multiple components.
  • statics - The statics object allows you to define static methods that can be called on the component class.

States

In React, you simply update a component's state, and then render a new UI based on this new state. React takes care of updating the DOM for you in the most efficient way. Read more about components and states here.

Composability

Since react promotes you to think of your UI in terms of pieces of functionality (components), you will obviously need multiple components.

Composition demonstrated: 


var UserProfile = React.createClass({
  render: function() {
    return (
      <div>
        <UserPic username={this.props.username} />
        <UserLink username={this.props.username} />
      </div>
    );
  }
});

var UserPic = React.createClass({
  render: function() {
    return (
      <img src={'https://graph.mysite.com/' + this.props.username + '/picture'} />
    );
  }
});

var UserLink = React.createClass({
  render: function() {
    return (
      <a href={'https://www.mysite.com/' + this.props.username}>
        {this.props.username}
      </a>
    );
  }
});

ReactDOM.render(
  <UserProfile username="sample user name" />,
  document.getElementById('entry-point')
);

The component "UserProfile" contains (owns) 2 sub components, a "UserPic" and a "UserLink".
